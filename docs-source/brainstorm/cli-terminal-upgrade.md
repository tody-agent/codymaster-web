## Qualified Problem

**For:** CodyMaster users (developers AND non-tech creators)  
**Who:** Need a friendly, intuitive terminal experience  
**The:** `cm` CLI is a project management + skill orchestration tool  
**That:** Should feel like talking to a smart, cute helper — not reading log output  
**Unlike:** Current state of plain chalk text and generic prompts  
**Our approach:** Transform CLI into a branded, animated, non-tech-friendly TUI with hamster mascot

### Root Causes
1. **Technical:** Monolithic `index.ts`, no UI component system, basic `console.log()` output
2. **Product:** No personality/brand identity in CLI, no user guidance system
3. **Design:** No layout engine, no animations, no responsive terminal sizing

### Impact if NOT addressed
- **Brand:** CodyMaster feels like a generic npm package, not a premium product
- **User:** Non-tech users bounce immediately — too intimidating
- **Market:** Falls behind Claude Code, Cursor, and other polished AI CLI tools

## Phase 3: Three Options

### Option A: 🎨 Chalk++ (Enhanced chalk + @clack/prompts)

**Approach:** Keep existing Commander.js architecture, replace `prompts` with `@clack/prompts` (already in devDeps!), add ASCII hamster, box-drawing characters, and spinners using chalk.

**Philosophy:** Minimal effort, maximum visual improvement.

**What changes:**
- `src/index.ts` → Refactor output functions to use box-drawing chars & `@clack/prompts`
- New `src/ui/` → Hamster ASCII art, box renderer, spinner, progress bar
- Banner → Hamster mascot with gradient welcome
- Tables → Replace `padRight()` with bordered tables
- Errors → Contextual help boxes with next-step suggestions
- Menus → Replace `prompts` with `@clack/prompts` (beautiful by default)

**Effort:** S-M (~2-3 days)  
**Risk level:** Low  

**Pros:**
- Zero new runtime deps (clack already in devDeps)
- Compatible with existing Commander.js structure
- Fast to ship
- No architectural rewrite needed

**Cons:**
- Still `console.log()` based — no real TUI (no live updates)
- Limited layout capabilities
- Can't do real-time dashboard in terminal
- Animations limited to stdout tricks

**Best when:** Quick win, ship improvement fast, iterate later.

---

### Option B: 🚀 Ink + React TUI (Full Rewrite)

**Approach:** Rebuild CLI renderer using Ink (React for terminal). Component-based UI with Flexbox layout, live updates, custom hamster component, and interactive widgets.

**Philosophy:** Claude Code-level quality. Modern, maintainable, extensible.

**What changes:**
- `src/index.ts` → Split into Commander command definitions + Ink renders
- New `src/tui/` → React components: `<HamsterMascot>`, `<StatusPanel>`, `<TaskBoard>`, `<SkillBrowser>`, `<ProgressBar>`, `<Spinner>`, `<Box>`, `<Welcome>`
- Full Flexbox layout → Headers, sidebars, footers
- Live updating dashboard in terminal
- Animated hamster mascot states (happy, thinking, working, celebrating)
- Interactive autocomplete (like Claude Code's slash commands)

**Effort:** L-XL (~5-8 days)  
**Risk level:** Medium-High

**Pros:**
- Professional-grade TUI like Claude Code
- Component reuse (React ecosystem)
- Live updating views
- Flexbox layout engine built-in
- Most extensible long-term

**Cons:**
- Heavy rewrite — breaks all existing output code
- Ink dependency adds weight (~1.5MB)
- React paradigm shift for CLI
- Testing TUI components is complex
- Slower startup than `console.log()`

**Best when:** Want to match Claude Code quality, long-term investment.

---

### Option C: 🐹 Hamster Shell (Hybrid — Chalk+Clack core + Ink for interactive views)

**Approach:** Keep Commander.js + chalk for simple commands (fast startup), use `@clack/prompts` for selections, BUT add Ink-powered "views" for interactive screens (dashboard, task board, skill browser). Hamster mascot throughout.

**Philosophy:** Best of both worlds — fast for simple, rich for complex.

**What changes:**
- `src/ui/core.ts` → Branded output helpers (boxes, tables, hamster art, spinners)
- `src/ui/hamster.ts` → ASCII hamster states + personality messages
- `src/ui/ink-views/` → Ink components ONLY for: Dashboard, Task Board, Skill Browser
- Simple commands (`cm status`, `cm task add`) → Enhanced chalk output (fast)
- Interactive commands (`cm`, `cm dashboard --tui`) → Ink-powered rich views
- `@clack/prompts` → Replace `prompts` for all selection menus
- New: Contextual help system ("Did you mean...?", "Try this next...")
- New: Hamster personality system (greeting based on time of day, encouragement on task done)

**Effort:** M-L (~4-5 days)  
**Risk level:** Medium

**Pros:**
- Fast startup for simple commands (no Ink overhead)
- Rich interactive views where they matter
- Hamster personality creates memorable brand
- Incremental — ship chalk improvements first, add Ink views later
- Non-tech friendly contextual help
- Uses `@clack/prompts` already in devDeps

**Cons:**
- Two rendering systems to maintain
- Slightly more complex architecture
- Need clear boundary between "simple output" and "Ink views"

**Best when:** Want quality AND speed, ship iteratively.
