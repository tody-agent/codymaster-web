**Approach:** Use an LLM agent script (`cm-dockit` variant) to scan the entire codebase locally, reading every file and using an AI prompt to write a 1-sentence "System Role" for each file (e.g. "This file handles the frontend routing", "This is the core Memory module"). **Effort:** M (~1 day)
**Pros:**
- Guarantees 100% human-readable context.
- Makes the code graph truly "intelligent".
**Cons:**

### Option C: **Hybrid Taxonomy "Lobes & Synapses" Model**
**Approach:** Map the project visually and semantically into "Brain Regions" using a predefined taxonomy file. For instance, any file under `skills/` is tagged `[Frontal Lobe: Decision Making]`. The 3D CodeGraph clusters them visually or assigns distinct organic colors per Lobe.
**Philosophy:** Business-first / Product-first (Extreme "WOW" factor and storytelling).
**Effort:** M (~1 day)
**Pros:**
- Delivers the exact "Brain Model" narrative the user wants to communicate.
- Requires no heavy LLM processing on every index run.
**Cons:**
- Less granular than Option B (files inside a Lobe share a generic context unless they have specific frontmatter).
