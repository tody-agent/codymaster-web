### SECTION 3: TECHNICAL DEEP-DIVE — "Under the Hood"

> 💡 *For the Technical Buyer — detailed specs*

#### 🔴 TDD Enforcement (`cm-tdd`)
```
AI MUST write tests FIRST → Red → Green → Refactor
No tests = no code. Simple as that.
```

#### 🛡️ 6-Gate Quality System (`cm-quality-gate`)
```
Gate 1: Static Analysis     → lint, type check, dead code
Gate 2: Blind Review         → AI reviews code WITHOUT knowing who wrote it
Gate 3: Security Scan        → dependency audit, SQL injection, XSS
Gate 4: Performance Check    → bundle size, O(n) analysis
Gate 5: Documentation        → JSDoc, README update check
Gate 6: Ship Gate            → final human approval
```

#### 🧠 Working Memory (`cm-continuity`)
```
Session 1: Build auth module → context saved to .cm/CONTINUITY.md
Session 2: "Continue" → AI reads full context → picks up EXACTLY where it left off
Session 3: "Continue" → still remembers, including MISTAKES from Session 1
```

#### 🤖 Judge Agent
```
🟢 CONTINUE  — task progressing normally
🏁 COMPLETE  — all subtasks done, ready for review
⚠️ ESCALATE  — stuck >10min, needs intervention
🔄 PIVOT     — 3+ failures → suggests alternative approach
```

#### 🔄 RARV Cycle (Autonomous Execution)
```
ORIENT   → Read CONTINUITY.md + past learnings
REASON   → Pick highest-priority subtask
PRE-ACT  → 🛡️ Goal alignment check (PREVENTS DRIFT)
ACT      → Execute with assigned skill
REFLECT  → Update working memory + learnings
VERIFY   → Quality gate → ✅ next task | ❌ retry (max 3)
```

### SECTION 9: FAQ — Dev-Specific

> 💡 *Technical objection handling*

**❓ "Does it slow down the pipeline?"**
> ✅ CodyMaster is a skills framework — 0 runtime overhead. Skills are just instructions for the AI. No extra API calls, no extra latency. Overhead = 0.

**❓ "Which languages/frameworks are supported?"**
> ✅ CodyMaster is agnostic. Skills work at the process level (TDD, quality gate, deploy) — it doesn't care if you use React, Vue, Python, Go, or Rust. If your AI tool supports it, CodyMaster supports it.

**❓ "I'm already using Cursor/Copilot. Is there a conflict?"**
> ✅ No. CodyMaster **runs ON TOP OF** the tools you already use. It adds a skills framework to Cursor/Copilot/Claude — it doesn't replace them. It's like adding ESLint to a project: no conflict, just added discipline.

**❓ "Where is the Working Memory stored? What about privacy?"**
> ✅ 100% local. `.cm/CONTINUITY.md` sits inside your project repo. No data is sent to any cloud. You can `.gitignore` it if you want — or commit it if the team wants to share context.

**❓ "I want to create a custom skill for my team. Is that possible?"**
> ✅ Of course. Create a folder `skills/your-skill/SKILL.md`, following the `cm-skill-mastery` template. The AI automatically loads and executes it. Team-specific coding standards? Conventions? Architecture patterns? Turn them all into skills.

**❓ "How does the Judge Agent work?"**
> ✅ The Judge Agent auto-evaluates task health. Stuck for >10 minutes? → ⚠️ ESCALATE. 3+ failures? → 🔄 PIVOT suggestion. Everything is displayed on the Dashboard. No need to babysit.
