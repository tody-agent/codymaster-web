Feature Pipeline

Building a new feature, step by step

Seven phases from idea to production. Each phase has a dedicated skill.

1

Strategic Analysis

cm-brainstorm-idea

Understand before solving. Multi-dimensional evaluation using Design Thinking, TRIZ, and Double Diamond.

2

Planning

cm-planning

Break into small testable tasks. Explore intent, requirements, and design before implementation.

3

Implementation

cm-execution + cm-tdd

Write test first, then code. Execute in thin slices. One behavior, one verification.

4

Code Review

cm-code-review

Spec compliance, security, quality. Full review lifecycle with technical rigor.

5

Quality Gate

cm-quality-gate

8-gate verification. Test gates, evidence-based checks, frontend safety. No claims without evidence.

6

Deployment

cm-safe-deploy

Safe ship with rollback. Multi-gate deploy pipeline with build verification and rollback strategy.

7

Monitoring

cm-status + dashboard

Track progress, monitor health, catch regressions early. Keep the pulse on your project.

Bug Fixing

Fix bugs permanently, not symptoms

Six steps to eliminate bugs at the root. No band-aids.

🔍

Reproduce

Write a failing test that demonstrates the bug. No fix without proof.

🦦

Root Cause

5 Whys analysis. Dig past symptoms to the actual origin. Keep asking why.

🧪

Regression Test

Lock the bug in with a test that will catch it if it ever returns.

🔧

Fix

Minimal, targeted change. Fix the root cause, not the surface manifestation.

✅

Verify

Run full test suite. Confirm the fix works and nothing else broke.

🛡️

Prevent

Update gates and lint rules so this category of bug can't happen again.

"Fix the root cause, not the symptom."
— cm-debugging philosophy
Judgment Calls

Skip the heavy workflow when…

Not everything needs a full pipeline. Use judgment.

✏️

Simple text changes

Typo fixes, label updates, copy edits. Just edit directly and commit.

⏱️

Quick one-off tweaks

If it takes less than 30 minutes and isn't user-facing, just do it.

📖

Documentation-only updates

README edits, comment cleanup, doc generation. Low risk, low ceremony.

📦

Dependency bumps

Automated version updates with passing tests. Trust the pipeline, not the process.
