# Proposal: Two-Zone Site Architecture
Generated by: cm-brainstorm-idea
Date: 2026-05-13

## Why we're doing this

### Qualified Problem
**For:** Developers & AI practitioners discovering CodyMaster  
**Who:** Land on `cody.todyle.com` and find a single page — no conversion funnel, no docs path  
**The:** CodyMaster web presence needs a two-zone architecture  
**That:** Guides visitors from awareness (landing pages) → understanding (docs)  
**Unlike:** Current state where docs are hidden and landing is a single monolith  
**Our approach:** Rebuild multi-page landing + wire VitePress docs with bidirectional navigation

### Root Causes
1. The 10-page funnel was collapsed into 1 HTML (`codymaster-landing.html`) during refactor
2. The "Guide" link was commented out in `header.html` — docs unreachable from landing
3. No design token bridge between landing CSS and VitePress theme

## What's changing

### Recommendation: Option B — Integrated Two-Zone with Shared Design System

**Zone 1 — Landing (`cody.todyle.com`):**
- 7-10 static HTML pages: `index.html`, `the-problem.html`, `the-solution.html`, `compare.html`, `skills.html`, `demo.html`, `get-started.html`
- Shared partials: `assets/partials/header.html`, `footer.html`
- `header.html` → uncomment Guide link, rename "Docs", point to `/docs/`

**Zone 2 — Docs (`cody.todyle.com/docs`):**
- Source: `docs-source/` markdown files
- VitePress base: `/docs/` (MUST keep for Cloudflare Pages)
- `docs-source/.vitepress/theme/custom.css` → brand token sync with landing colors

**Cross-zone bridge:**
- Landing header → "Docs" link → `/docs/`
- VitePress nav → "Website ↗" → `https://cody.todyle.com`
- VitePress home → "Get Free" CTA → `https://cody.todyle.com/get-started.html`
