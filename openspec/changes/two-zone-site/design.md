# Design: CodyMaster Two-Zone Site — Landing + VitePress Docs
Initiative: two-zone-site
Generated by: cm-planning
Date: 2026-05-13

## Context & Technical Approach

### Current State
- **Zone 1 (Landing):** 1 HTML file (`codymaster-landing.html`) with inline header/footer (NOT using partial injection). Nav links point to `the-problem.html`, `the-solution.html`, `demo.html`, `workflow.html`, `get-started.html` — these pages do NOT exist yet.
- **Zone 2 (Docs):** `docs-source/` → VitePress 1.6, `base: '/docs/'`, builds to `docs/`. Working but unreachable from landing (Guide link commented out).
- **Shared design system:** `assets/css/base.css` (tokens), `assets/partials/header.html` + `footer.html`, `assets/js/components.js` (partial injector).
- **Problem:** Landing is a monolith; no funnel pages exist; two zones are disconnected.

### Chosen Approach: Option B — Integrated Two-Zone
1. **Phase A:** Create 7 HTML landing pages using the partial injection pattern (not inline header/footer like the monolith).
2. **Phase B:** Bridge the two zones (header Docs link + VitePress theme token sync).
3. **Phase C:** Rename `codymaster-landing.html` → `index.html` and make it the homepage.

### Key Technical Decisions
- **Partial injection pattern:** All new pages use `<div id="cm-header-mount"></div>` + `<div id="cm-footer-mount"></div>` + `components.js` (already working).
- **Page identity:** Each page sets `<body data-page="...">` for active nav highlighting.
- **VitePress base:** MUST stay `/docs/` — do NOT change (Cloudflare Pages deploy constraint).
- **Content source:** Extract sections from `codymaster-landing.html` for reuse, then expand.
- **index.html:** The monolith becomes `index.html` — strip inline header/footer, replace with mount divs + inject via components.js.
