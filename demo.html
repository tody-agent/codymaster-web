Help

Common issues

Quick fixes for the most frequent setup problems.

"cm: command not found"
Check that npm's global bin directory is in your PATH. Run npm config get prefix and add /bin to your shell profile.
"Configuration file not found"
Run cm init in your project root to generate the .cm/ config directory with default settings.
"MCP server failed to start"
Check if the default port is already in use. Run lsof -i :3845 and kill the process, or use cm mcp-serve --port 3846.
"Git hooks not working"
Run cm hooks install in your project root. This sets up pre-commit and commit-msg hooks.
"Quality gate failing"
Run cm gate --verbose to see detailed output. Most failures are missing test files or broken imports.
