Common Mistakes

How to Talk So AI Actually Gets It

Good prompts = good results. Here are the most common mistakes and how to fix them.

Too Vague

❌ Wrong

"Make me a beautiful website"

✅ Right

"Create a landing page for a yoga studio called ZenFlow. Pastel tone, light purple + white. Need intro section, 3-tier pricing, and a trial signup form."

Wanting Everything at Once

❌ Wrong

"Build a complete app with everything: login, management, reports, payments, chat..."

✅ Right

"Step 1: Create the login and registration page. Once this is done, we'll move on."

Not Reviewing the Plan

❌ Wrong

AI shows plan → You: "OK do it" (without reading)

✅ Right

Read the plan → Give feedback: "Remove chat, add a gallery section" → AI revises → Then approve.

Ignoring the Design Review Step

❌ Wrong

Skip the mockup preview — jump straight to code

✅ Right

Review the design preview → say "make the header bigger, change CTA to green" → then approve for coding.

No Reference Examples

❌ Wrong

"Make it look professional" (what does "professional" mean to you?)

✅ Right

"Make it look like Stripe's homepage — clean, minimal, lots of white space." Or send a screenshot URL.
Pro Tips

7 Tips for Better Vibe Coding

🎯

Be Specific

Say: brand name, colors, how many sections, what features. The more specific, the more accurate the AI.

🪜

Work Step by Step

Don't ask for a "complete app" right away. Break it down: homepage → form → dashboard → deploy.

👀

Review the Plan

AI always shows a plan before coding. Read and give feedback. This is your most important "director" moment.

🗣️

Speak Your Language

CodyMaster understands Vietnamese, English, and more. No need for technical jargon or perfect English.

📸

Share References

Have a website you like? Send the URL or a screenshot. AI will analyze it and create a similar design.

🔄

Iterate Continuously

"Make the CTA button bigger", "Change the font", "Add a testimonials section". Chat naturally to refine.

💾

Use /cm-start

Type /cm-start "project description" and CodyMaster automatically runs the entire workflow for you.

Frequently Asked Questions

Got Questions? We've Got Answers

I don't know any code at all. Can I still use this?
Absolutely! The creator of CodyMaster (Tody Le) isn't a developer either — he's a Product Manager. CodyMaster is designed so you only need to chat in natural language. AI handles all the technical work.
How is this different from copy-pasting code from ChatGPT?
ChatGPT gives you scattered code snippets — you have to stitch them together, test, and deploy yourself. CodyMaster is a complete system: 68+ specialized skills automatically run — from planning, design, coding, testing, security scanning, to deployment. It's like hiring an entire senior team instead of asking one random person.
What do I need to install before starting?
You need one of these AI coding agents: Cursor IDE, Claude Code, Gemini CLI, or any agent that supports skills. Then run one install command for CodyMaster and start chatting. See the step-by-step guide on the Get Started page.
How much does it cost?
CodyMaster is completely free and open source. You only pay for the AI agent you use (e.g., Cursor subscription or Claude API). Many agents have free tiers that are enough to get started.
What if I want to make changes later?
Just chat again! CodyMaster has "memory" (cm-continuity) so it remembers your project context. Just say: "Change the background to navy blue" or "Add a testimonials section" — AI updates without breaking existing code.
What kinds of products can I build?
Landing pages, blogs, e-commerce stores, dashboards, booking systems, SaaS apps, portfolios, technical documentation, content marketing... any web product. CodyMaster even supports mobile app UI design.
