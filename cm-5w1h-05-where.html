MCP Protocol

The universal bridge

Model Context Protocol connects CodyMaster to any AI client that supports it. One plug, many devices.

What it is

A standard protocol for connecting AI agents to external tools and data sources. Think of it as USB for AI — one plug, many devices.

Why it matters

Any MCP-compatible client can access CodyMaster's full tool suite. No custom integrations needed — just point and connect.

18 MCP tools exposed including:

cm_memory_read cm_memory_write cm_memory_search cm_context_get cm_context_advance cm_budget_check cm_skill_resolve cm_natural +10 more

Claude Desktop — JSON

{
  "mcpServers": {
    "codymaster": {
      "command": "npx",
      "args": ["codymaster", "mcp"]
    }
  }
}

Installation

Three ways to install

Pick the method that fits your workflow.

Recommended
Method 1

Global Install

npm install -g codymaster && cm

The wizard detects all AI tools on your system and configures them automatically. Best for most developers.

Method 2

Project-Level

npm install --save-dev codymaster

Per-project setup. Skills and config live inside your repo. Great for teams that want version-controlled AI tooling.

Method 3

One-Line Script

curl -fsSL https://codymaster.dev/install.sh | bash

For systems without Node.js. Downloads a standalone binary and sets up everything for you.

Setup Wizard

The wizard handles everything

Five steps from zero to fully configured.

1
Detects all AI coding tools installed on your system
2
Creates skill configuration for each tool
3
Sets up quality gates and git hooks
4
Initializes memory system (CONTINUITY.md)
5
Runs verification (cm doctor)
